- feature 子域以目录命名（如 `daily-review/`、`mission/`）并在其下导出一个主 Panel 组件，由 `main.tsx` 中的 `AppSection` 联合类型集中分发渲染。
- TiPTAP 相关代码按职责拆分为 `tiptap-node`（自定义节点实现+SCSS）、`tiptap-ui`（操作按钮与 hook）、`tiptap-icons`（lucide 风格 SVG 组件）、`tiptap-ui-primitive`（button/badge 等原子控件）四个子目录。
- UI 组件统一采用函数组件 + 独立 `.scss` 样式文件，并使用 `clsx` 组合 className，避免内联 style。
- 全局副作用与平台能力通过 `declare global { interface Window }` 进行类型声明后再访问（如 `aistudyClipboard`），而非直接 `any` 强转。
- 测试文件与源码同目录放置，命名为 `*.test.ts(x)`，并通过 `vitest.config.ts` 的 include 规则统一发现，不使用全局 expect。
