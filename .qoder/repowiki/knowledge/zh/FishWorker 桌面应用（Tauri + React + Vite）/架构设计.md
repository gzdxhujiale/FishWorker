根目录为单仓前端工程，通过 `pnpm-workspace.yaml` 仅声明 watcher 构建白名单；入口 `index.html` 加载 `/src/main.tsx`，由 `AppErrorBoundary` 包裹根组件，使用 `useState<AppSection>` 在 `main.tsx` 内按路由键分发到各 feature 面板。

- 构建与运行：Vite 7 + `@vitejs/plugin-react`，`vite.config.ts` 固定端口 1420/1421 HMR，并通过 alias 将 `quill/*` 重定向到本地 `scripts/npm-stubs/quill` 以绕过 `simple-mind-map` 对 quill 的依赖冲突；`tsconfig.json` 配合 `tsc -b` 做增量编译。
- 测试：`vitest.config.ts` 配置 jsdom 环境，匹配 `src/**/*.test.{ts,tsx}`。
- 源码分层（位于 `src/`）：
  - `components/`：跨功能 UI 组件，细分为 `layout/`（`AppLayout`、`MenuBar`、`Toolbar`）、`tiptap-*`（编辑器节点、图标、UI 控件、primitive 基础按钮/徽章等），全部为 React 函数组件 + SCSS/CSS Modules 样式。
  - `features/`：业务特性目录，包含 `daily-review`、`habits`、`lists`、`mission`、`settings`、`time-management`、`tiptap` 七个子域，每个子域自包含面板/弹窗与状态逻辑。
  - `hooks/`：可复用 Hook（滚动、断点、键盘快捷键、TiPTAP 编辑器封装等）。
  - `lib/`：纯函数库，如 `createSyncEngine` 同步引擎与 TiPTAP 工具集，附带单元测试。
  - `styles/`：全局 SCSS 变量、动画 keyframe、布局与对话框样式。
  - `types/global.d.ts`、`vite-env.d.ts`：全局类型扩展。
- 运行时边界：通过 `@tauri-apps/api`、`plugin-opener`、`plugin-window-state` 与 Rust 后端通信；`Window.aistudyClipboard` 作为外部剪贴板桥接的可选 API。
