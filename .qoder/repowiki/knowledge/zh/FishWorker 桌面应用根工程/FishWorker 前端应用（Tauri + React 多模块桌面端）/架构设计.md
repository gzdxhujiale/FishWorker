入口 `src/main.tsx` 使用 React 18 `createRoot` 挂载根组件，并用类组件 `AppErrorBoundary` 包裹全局错误恢复；`AppLayout`（`components/layout/`）提供 Tauri 原生菜单栏（`@tauri-apps/api/window`）、左侧工具栏与主内容区三个插槽，由 `main.tsx` 根据 `activeSection` 状态渲染对应 Feature 面板。

Feature 层按业务域拆分为 `features/{lists,time-management,daily-review,habits,settings,tiptap}`，每个 Feature 内部遵循统一的三层结构：`*Store.ts`（Zustand store，定义 state + actions + init 生命周期）、`*Service.ts`（I/O 封装，含 TiDB 同步与 localStorage 降级）、`*Types.ts`（类型定义），UI 组件仅消费 store 并通过 service 触发持久化。

持久化采用乐观更新 + 去抖同步：store 内调用 `lib/createSyncEngine.ts` 暴露的 `schedule(key, persist)` 对同一 key 的写操作进行 debounce（默认 500ms），删除时调用 `cancel(key)` 防止竞态；`createSyncEngine` 是纯函数工厂，不持有外部依赖，仅维护 `Map<string, number>` 定时器表。

富文本编辑器以 `components/tiptap-*` 子目录组织为可复用库：`tiptap-ui-primitive` 提供 button/badge/popover 等基础控件，`tiptap-ui` 组合出带 hook 的编辑按钮（`use-mark.ts`、`use-list.ts` 等），`tiptap-node` 扩展 Blockquote/CodeBlock/Image 等节点，`tiptap-templates/simple` 给出最小可用示例；`features/tiptap` 则把上述能力装配成 SlashCommands、BubbleMenu 等高级特性。

样式体系：全局变量与动画集中在 `styles/_variables.scss`、`_keyframe-animations.scss`，各 Feature 自带 `.css` 文件，TiPTAP 相关组件用 `.scss` 局部覆盖主题。
