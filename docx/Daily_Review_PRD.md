# 产品需求文档 (PRD) - 每日复盘模块 v1.0

## 1. 概述
### 包含:
- **问题陈述：** 用户需要记录每日的学习、反思及改进计划，以实现长期的习惯养成和能力复利增长。当前的编辑体验较为单一（普通 textarea 纯文本输入框），缺乏格式排版（加粗、列表、标题等）和流畅的富文本输入支持，限制了内容的表达与整理。此外，当前的高频输入同步机制在每次按键时直接保存，容易对后端数据库产生频繁写入的压力，需进一步优化性能。
- **建议的解决方案：** 为“每日复盘”模块集成 TipTap 富文本编辑器，支持 Markdown 语法快捷转换与常用富文本格式。在数据流动与保存上，引入“前端内存状态乐观更新 + 延迟防抖异步持久化”机制（打字时零延迟响应，停止输入 500ms 后自动异步保存至云端），并提供同步状态提示（同步中、已同步、同步失败），确保极致流畅且性能优异的编辑体验。

## 2. 核心理念
- **复利成长与习惯打卡 (Streaks & Compound Growth)**：通过连续记录天数（Streak）和数值化的“复利成长指标”（如 $1.01^{365} \approx 37.8$），给予用户视觉上的正向反馈与成就感，激励用户每日坚持复盘。
- **结构化深度记录 (Structured Reflection)**：通过 TipTap 编辑器，用户可以利用粗体、列表、引用块和任务列表，将反思内容组织得更为清晰，提升复盘质量。
- **零阻塞交互 (Optimistic & Fluid UI)**：任何编辑操作均在前端内存中瞬间完成更新，而向后端的网络/IPC 持久化由防抖定时器异步执行，绝不阻塞用户正常的键入流程。

## 3. 解决方案概述

### 3.1 建议的解决方案
- **高层级描述**：
  “每日复盘”页面分为左侧的复盘数据统计与日历打卡区，以及右侧的富文本复盘编辑区。整个页面拥有独立的每日打分、天数切换以及同步云端的状态展示。
- **核心能力**：
  - **成长与统计卡片**：展示当前复利值、总复盘天数、当前连续打卡天数和历史最高连续天数。
  - **打卡热力图**：按月份展示打卡记录，不同分数段（1-5星）对应不同的视觉高亮热力状态，支持点击日历日期快速加载对应日期的复盘。
  - **TipTap 富文本编辑器**：替换传统的 Textarea，支持 Markdown 语法解析（如 `# ` 变标题，`* ` 变无序列表等）、富文本修改与 placeholder 引导语。
  - **乐观更新与防抖保存**：打字时前端状态立刻更新，并在用户停止修改 500ms 后自动触发 Tauri 的 Rust API 将数据异步保存至云端。
  - **同步云端状态指示**：在编辑区域右上角实时指示当前数据的保存状态（“同步中...”、“已同步”、“同步失败”）。

### 3.2 包含在本次范围内
- **功能点 1：成长统计与打卡热力图 (CompoundStats)**
  - 根据连续记录天数自动计算复利值并高亮展示。
  - 渲染当月的打卡记录（使用 `react-day-picker`），每一天依据当天的评分（Rating：1-5星）显示不同程度的高亮徽标。
  - 支持通过日历日期选择、以及编辑区顶部的“前一天 / 后一天”按钮快捷流转日期，不可跨入未来日期。
- **功能点 2：TipTap 富文本编辑器集成 (ReviewEditor)**
  - 在 `ReviewEditor` 中使用 `@tiptap/react` 替换原有的 textarea 元素。
  - 支持配置 placeholder（例如：“今天学到了什么？什么可以改进？明天最重要的一件事？...”）。
  - 支持基本富文本操作（如粗体、斜体、列表、任务清单等）和 Markdown 快捷语法的自动渲染。
- **功能点 3：前端内存状态乐观更新与高频输入防抖 (Optimistic & Debounced Saving)**
  - 状态管理：编辑器的内容（content）发生变更时，前端优先更新内存中的 review 列表缓存，保持界面的响应速度。
  - 防抖计时器：当用户开始输入时，拦截立即持久化；当用户停止输入 `500ms` 后，再触发 `list_upsert_daily_review` 异步 IPC 指令。
  - 对于低频交互（如评分选择 1-5 星），不进行防抖，立即触发同步。
- **功能点 4：数据持久化**
  - 后端对接：每次防抖时间到达或点击星级评分时，调用 Tauri 命令，连接远端 TiDB 数据库将复盘记录进行 upsert。
- **功能点 5：无显式删除与自动判定删除**
  - 移除了显式的“删除复盘”按钮。
  - 当一天的复盘内容（TipTap 富文本 HTML）清空或为空时，判定为“当天没有复盘”。
  - 当用户清空编辑器内容，防抖 500ms 后自动触发后端 API 删除对应的复盘记录，同时更新本地状态，使得打卡日历中该日期的热力点熄灭、复盘统计数据及连续打卡天数重新计算。

### 3.3 超出本次范围
- 一次性导出所有日期的复盘记录生成长文档或 PDF 报告。
- 内置 AI 复盘助手（如根据内容自动分析当日收获并进行智能评级）。

## 4. 用户故事与需求

### 4.1 用户故事
```text
作为一名 寻求长期成长的自我管理者
我希望 在每天结束时在一个流畅、美观的编辑器里写下复盘内容
以便于 我可以随心所欲地使用列表和加粗排版，并且能在打字时不需要等待数据库保存的网络延迟，随时看到数据已成功同步的绿云标志。

验收标准：
[ ] 进入每日复盘页面，右上角显示“已同步”状态。
[ ] 在编辑器内快速键入字符时，编辑器响应完全流畅无延迟。右上角状态转为“同步中...”。
[ ] 停止打字 500ms 后，右上角重新变回绿色的“已同步”。
[ ] 用鼠标点击打分星级，评星立刻变化，且同步状态立即变为“同步中...”，保存完毕后恢复“已同步”。
```

### 4.2 功能需求
| ID | 需求描述 | 优先级 | 备注 |
|----|------------|---|-------|
| FR1 | 页面多面板展示 | P0 | 左侧展示复利成长指标与打卡日历热力图；右侧展示编辑器与评星区域 |
| FR2 | 日历评分热力点映射 | P0 | 绑定 `react-day-picker`，将各天 rating（1-5）划分为四种渐变绿/蓝背景点表示打卡频度与状态 |
| FR3 | 日期导航控制 | P0 | 支持“前一天/后一天”按钮切换，且“后一天”不能超过系统当前日期 |
| FR4 | TipTap 编辑器嵌入 | P0 | ReviewEditor 中载入 TipTap，具备 Markdown 语法实时转换与多格式支持 |
| FR5 | 内存状态乐观更新 | P0 | 用户键入内容时，直接保存至本地 state 中并反映在日历中，不发生卡顿 |
| FR6 | 编辑防抖持久化 | P0 | 用户编辑内容时，防抖防爆触发延迟设置为 `500ms`，停止编辑 500ms 后方将内容发送至 Tauri 后端进行云端持久化 |
| FR7 | 隐藏同步状态可视化指示 | P1 | 不提供“同步中...”/“已同步”/“同步失败”的视觉指示 |
| FR8 | 自动判定与内容级删除 | P0 | 移除显式“删除复盘”按钮。当复盘文本内容清空时，自动同步删除该记录，不计入总复盘天数及连续打卡统计。 |

### 4.3 非功能需求
- **交互流畅度**：打字时的响应延迟必须在 `16ms` 内（即 60FPS），不能因为后台网络写入或 IPC 同步导致编辑器输入卡顿或跳字。
- **防抖时效**：`500ms` 的防抖时间应精确，如果在防抖期内用户切换了复盘日期，需立刻对前一日期的修改执行“强制同步”，避免数据丢失。

## 5. 设计与用户体验

### 5.1 设计原则
- **极简而高级**：采用深色毛玻璃或柔和卡片布局，使用墨绿色/青色系列体现复利成长的“勃勃生机”。
- **状态不打扰**：同步状态图标仅在右上角以小字或云状小图标展示，确保用户能保持沉浸式写作“心流”。

## 6. 技术规格说明

### 6.1 前端核心组件列表
1. **`DailyReviewPanel.tsx`**：主入口容器，同步数据库，向下分发 `reviews` 数据和当前选中的 `selectedDate`。
2. **`CompoundStats.tsx`**：接收复盘列表，进行打卡天数计算，并渲染 `DayPicker`。
3. **`ReviewEditor.tsx`**：内置 TipTap 实例（`useEditor`），绑定 `onUpdate` 事件，并通过 React Effect 驱动 500ms 防抖更新。

### 6.2 数据模型定义 (参考)
```typescript
export interface DailyReview {
  id: string;      // 格式: review-YYYY-MM-DD
  date: string;    // YYYY-MM-DD
  content: string; // HTML 格式的 TipTap 复盘文本
  rating?: number; // 1-5 的分数值
  createdAt: number;
  updatedAt: number;
}
```

### 6.3 乐观更新与防抖实现说明
- 前端内存乐观更新：当富文本编辑内容或评分改变时，即时更新对应的 UI 状态（如打卡日历点高亮与连续打卡天数）。
- 防抖时间设置：编辑器内容改变时启动 `500ms` 的计时器防抖，避免每次击键都保存；如果在计时器未到时用户直接切换了日期，系统会取消当前防抖计时并**立刻强制执行**当前已修改的数据同步，以防止未保存的内容丢失。
