- PRD 文档统一遵循“概述 → 核心理念 → 解决方案概述 → 包含/超出范围 → 用户故事(含验收标准) → 功能需求表格(ID/描述/优先级/备注) → 非功能需求 → 设计原则与页面结构 ASCII 图 → 技术规格(组件列表 + TS Interface)”的固定章节顺序。
- 接口文档以 TypeScript Interface / type 代码块作为唯一权威数据契约，供 AI 直接复制生成代码，避免自然语言歧义。
- 时间敏感型交互在 PRD 中明确量化指标（如“500ms 防抖”、“16ms 响应延迟”），并在非功能需求段单独列出可测试的验收阈值。
- 版本化演进通过文件名后缀区分（如 `time_management_prd_v1.1.md`、`v1.2.md`），增量变更以“[v1.x 更新]”标注而非覆盖旧文档。
