# Time Management UI Update PRD

## 1. 概述
- **问题陈述：** 当前时间管理模块的 UI 布局存在一些不够灵活和清晰的地方。周计划看板的高度未能自适应占满，每天没有明确的时间段划分；四象限工作台（思想性工作台）与周计划共用了侧边栏（待办分类/角色），导致空间浪费；四象限任务的添加入口不统一，且任务卡片显示过于复杂。
- **建议的解决方案：** 优化本周计划看板的高度自适应和上下午分区，并精简任务上的时间显示；在切换到四象限工作台时隐藏左侧边栏；重构四象限任务添加方式为象限右上角的快速弹窗；引入基于截止时间的折叠标签功能来分类展示任务；同时优化四象限卡片样式（单线分隔、去拖拽把手）以及周计划与四象限之间的数据隔离与同步逻辑。

## 2. 核心理念
通过更加合理的空间布局和任务分类展示，提升用户在时间管理模块中的操作效率与信息获取的直观性。消除无关信息的干扰（去除冗余标签、拖拽图标等），实现更加沉浸的任务管理。明确“本周计划”与“四象限”定位的差异。

## 3. 解决方案概述

### 3.1 建议的解决方案
- **高层级描述**: 优化 FishWorker 时间管理模块的界面布局与交互体验，主要涵盖本周计划看板和四象限（思想性）工作台的视图与数据流重构。
- **核心能力**:
  - 本周计划看板的自适应高度与上下午分区结构，及拖拽自动设置截止日期与Q2属性机制。
  - 四象限工作台的沉浸式大屏视图（动态隐藏侧边栏）。
  - 四象限任务的场景化快速弹窗添加，及极简卡片样式设计。
  - 四象限任务基于截止时间维度的标签化分组与折叠。
  - 明确的数据范围隔离：四象限内创建的任务默认不进入本周计划看板。

### 3.2 包含在本次范围内
- **功能点 1：本周计划看板样式与交互优化**
  - 将原“周计划看板”更名为“本周计划看板”。
  - 列（Column）高度自适应，占满剩余可用空间。
  - 每天的工作区划分为“上午”和“下午”两个独立区域。
  - 移除本周计划看板中任务卡片上的时间截止标签，保持界面整洁。
  - **核心逻辑**：在周计划的任务拖拽自动增加那天的截止时间，且该任务会自动被归入四象限工作台的“重要不紧急” (Q2) 象限内。
- **功能点 2：四象限工作台侧边栏显隐逻辑**
  - 当切换至“四象限工作台” Tab 时，隐藏时间管理内容区左侧的“待办分类/角色栏”。
- **功能点 3：四象限任务快速添加入口与弹窗**
  - 在每个象限面板的右上角新增“+”添加任务按钮。
  - 点击按钮后弹出快捷输入浮层（包含输入框、日期选择、优先级标记等）。
  - **核心逻辑**：四象限内创建的计划不会显示在本周计划看板内。
- **功能点 4：四象限任务卡片自定义极简样式**
  - 采用自定义组件设计，任务之间仅采用简单的水平横线进行间隔。
  - 任务保持可拖拽的交互能力，但移除界面上的“可拖拽图标”（Grip把手），使界面更加清爽。
- **功能点 5：四象限任务按截止时间分组显示**
  - 视图层将相同象限内的任务按照截止日期自动分为以下可折叠标签组：
    - 【无日期】
    - 【一天内】
    - 【三天内】
    - 【一周内】
    - 【一周外】
  - 标签支持点击折叠/展开。

### 3.3 超出本次范围
- 复杂的时间线（Gantt）视图。

## 5. 设计与用户体验

### 5.1 设计原则
- **极简主义：** 去除不必要的装饰线、背景色和 Icon（如拖拽把手、时间标签），用留白和简单的线条区分信息层级。
- **职责单一：** 明确“本周计划”是用于宏观的时间槽安排（自动落入Q2），而“四象限”是全局任务池。四象限新建的任务不反向污染周看板。

### 5.2 页面结构与交互流程
- **本周计划看板页面：**
  - Column 结构 -> `[Morning Area]` / `[Afternoon Area]`。
  - 卡片内部只显示 Title 和 Role 颜色标记。
- **四象限工作台页面：**
  - **结构**：`[Quadrant Q1] [Quadrant Q2]` ...
  - **象限任务样式**：
    - 去除明显的 Card 边框，使用 `border-bottom: 1px solid var(--line-soft)` 分隔任务项。
    - 隐藏 `:hover` 时的 Grip 图标，利用整个任务条目响应 HTML5 Drag 事件。
- **数据流转边界：**
  - 从左侧待办拖入本周计划 -> `deadline`=当日, `quadrant`='Q2', `scheduledDate`=当日。
  - 在四象限面板创建 -> `quadrant`=当前面板, 不设定 `scheduledDate` (不在周计划显示)。

### 6. 技术规格说明

#### 6.1 前端组件变更建议
- `WeeklyPlanning.tsx`: 
  - 更改头部文案，移除渲染卡片时的 `<Clock>` 时间标签代码。
  - `handleDrop` 中补充逻辑：除了设置 `scheduledDate`，同时更新 `deadline` 和 `quadrant: 'Q2'`。
- `DailyQuadrants.tsx`: 
  - 任务渲染方法（如 `renderTasks`）中移除 `<GripVertical>` 组件。
  - 修改对应的 CSS 类，去除边框和背景，改用底边线分隔。
  - 数据源过滤：确保只在四象限组件中显示这些任务，而 `WeeklyPlanning` 只抓取带有 `scheduledDate` 的任务（四象限新建默认无 `scheduledDate`，已满足需求）。
