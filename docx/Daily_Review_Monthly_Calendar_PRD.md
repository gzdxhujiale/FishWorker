# Product Requirements Document: 每日复盘当月打卡日历视图重构

## 1. 概述
- **问题陈述：** 目前每日复盘模块使用的是“最近 35 天”的无日期热力图（Heatmap），由于下方没有标注具体日期，且只显示颜色区块，导致用户无法直观辨认每天对应的打卡状态，体验不佳。
- **建议的解决方案：** 将“最近 35 天打卡记录”重构为“当月打卡记录（Monthly Calendar View）”，引入成熟的第三方日历组件（推荐 `react-day-picker`），在格子内直接展示日期数字，并结合打卡质量（评级）展示不同的背景颜色。同时确保前后端数据格式在日历组件间的转换严谨正确。

## 2. 核心理念
打卡日历应当提供直观的时间锚点。相较于类似于 GitHub 的纯色块热力图，带有明确日期数字的月视图日历更符合传统用户对“每日复盘”打卡行为的认知习惯，能够大幅增强连贯性和获得感。

## 3. 解决方案概述

### 3.1 建议的解决方案
- **高层级描述**: 在每日复盘页面的左侧，将原有的热力图替换为真实的月视图日历。日历展示当前月份，并支持跨月切换。
- **核心能力**:
  - 月视图日历展示（包含周一到周日的表头）。
  - 日期格子内显示具体号数（1, 2, 3...）。
  - 日历格子背景色根据当天的打卡等级（rating 1-5）应用相应的绿色高亮（与原有的 active-level-1~4 对应）。
  - 点击日历上的特定日期，右侧复盘编辑器联动切换至该日期。

### 3.2 包含在本次范围内
- **功能点 1：** 引入并集成 `react-day-picker` 及配套日期处理库 `date-fns`。优先级：高
- **功能点 2：** 重构 `CompoundStats.tsx`，移除旧的 35 天 heatmap 代码，替换为 Monthly Calendar 组件。优先级：高
- **功能点 3：** 确保日期数据格式转换的安全性。确保后端返回的 `YYYY-MM-DD` 和前端的本地时区 `Date` 对象在日历高亮和交互时不会发生时区偏移（Off-by-one day bug）。优先级：高
- **功能点 4：** 为日历提供与当前应用护眼配色匹配的自定义 CSS 样式。优先级：中
- **功能点 5：** 日期最早是2026-01-01，日历图不显示2026年之前的日期，日历可以点击上个月和下个月切换，不能显示、点击2026年之前的日期。
### 3.3 超出本次范围
- 支持查看过去一整年的打卡统计聚合数据分析。
- 生成包含日历的分享海报图片。

## 4. 用户故事与需求

### 4.1 用户故事
```text
作为一名 正在进行复盘的用户
我希望 能够在一个带有具体日期数字的月历上看到我的打卡记录
以便于 我能直观地知道自己哪几号完成了复盘、哪几号遗漏了，而不是去数没有日期的色块。

验收标准：
[ ] 日历正常显示当月所有的天数及对应的日期数字。
[ ] 已经复盘的日期会根据复盘评分显示不同深浅的绿色背景。
[ ] 今天的日期有特殊标识（如特定文本颜色或加粗）。
[ ] 选中的日期会有显眼的选中边框。
[ ] 点击任意有效日期，右侧编辑器能够正确切换到那一天。
[ ] 日历渲染和高亮不会受时区偏差影响而发生前移或后移一天的错位。
```

### 4.2 功能需求
| ID | 需求描述 | 备注 |
|----|------------|-------|
| FR1 | 系统需要引入 `react-day-picker` 组件 | Critical for MVP |
| FR2 | 日历初始渲染时应显示 `selectedDate` 所在的月份 | Critical |
| FR3 | 日历应支持通过头部导航按钮切换上/下个月 | Important |
| FR4 | `reviews` 数组中的数据必须准确映射为日历修饰符（Modifiers）进行差异化渲染 | Critical |

### 4.3 非功能需求
- **易用性 (Usability)：** 日历的数字必须清晰可读，颜色区块与文本的对比度要足够高。
- **可靠性 (Reliability)：** 必须在解析与格式化日期时采用统一的方法（如 `date-fns` 的格式化方法或手动本地时区切割），彻底杜绝跨天导致的高亮错误。

## 5. 设计与用户体验

### 5.1 设计原则
- **原则 1：** 数据清晰。背景颜色代表完成度和质量，不干扰日期的数字阅读。
- **原则 2：** 交互一致。日历的选择行为与顶部的日期切换按钮（`<` 和 `>`）逻辑互通，状态保持完全同步。

### 5.2 页面结构与交互流程
- 组件位置：依然位于 `DailyReviewPanel` 布局的左侧卡片区。
- 用户不仅可以通过点击右侧的“昨天”、“明天”箭头切天，也可以直接在左侧点选日历进行跨天、跨周跳转。

### 5.3 信息架构
- 数据来源：`stats` 属性以及 `reviews` 列表。
- 将 `reviews` 加工为一个按等级映射的 `Map` 或者特定结构对象，注入日历组件作为 `modifiers`。

## 6. 技术规格说明

### 6.1 库选择与安装
建议使用以下命令安装所需依赖：
`pnpm add react-day-picker date-fns`
*(注：`react-day-picker` v8+ 支持极其灵活的自定义渲染，无需引入默认的重度 CSS，完美契合当前应用的自定义 UI 体系。)*

### 6.2 数据处理与转换逻辑
由于前端当前的核心状态 `selectedDate` 是 `YYYY-MM-DD` 格式的字符串：
1. **日历初始焦点**：`month={new Date(y, m-1, d)}`
2. **高亮判断 (Modifiers)**：
   提取出复盘中 `rating` 不同的记录集合，分配给 `level1`, `level2`, `level3`, `level4` 等 modifiers。
3. **点击回调 (onDayClick)**：
   捕捉组件抛出的原生 `Date` 对象，调用如 `format(date, 'yyyy-MM-dd')` 或自定义函数格式化后触发 `onSelectDate`，保障无时区损失地将数据回流至全局状态。
