本模块为纯文档仓库，按功能域组织 Markdown 文件：
- PRD 类：`Daily_Review_PRD.md`、`Four_Quadrants_Workdesk_PRD.md`、`Weekly_Planning_PRD.md`、`time_management_prd_v1.1.md`/`v1.2.md`、`Zustand_Migration_PRD.md`、`UI_Refactoring_PRD.md`、`Tiptap_Simple_Editor_Template_PRD.md`、`list_feature_prd.md` 等，统一采用“概述→核心理念→解决方案→用户故事→功能需求表→非功能需求→设计与交互流程→技术规格（组件列表 + TS Interface）”的结构。
- API 规范类：`Frontend_Interfaces.md`、`Lists_Frontend_API.md`、`Time_Management_API_Doc.md`、`Time_Management_API_Spec.md`、`Weekly_Planning_Frontend_API.md`、`Zustand_API_Interface_Doc.md`、`习惯API接口文档.md`，以 TypeScript Interface 形式定义跨模块共享的数据类型与组件 Props，作为 AI 编码的 Source of Truth。
- 实践与复盘类：`FishWorker 项目复盘与总结报告.md`、`Matt_Pocock_Skills_说明文档.md`、`Typora_Hook_Log.txt`，记录开发方法论与技术债演进。
所有文档通过 `Frontend_Interfaces.md` 中定义的 `ToolConfig` 等全局接口形成横向契约，使各 PRD 能像插件一样被 AI 集成到现有架构中。
