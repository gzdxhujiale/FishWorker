# Product Requirements Document (PRD) Templates

## Standard PRD Template

### 1. 产品概述

#### 包含:
- **背景与问题陈述：** （2-3 句话）
- **产品目标：** （2-3 句话）

- **高层级描述**:  我们正在构建的产品/功能是什么（2-3 句话）
- **核心能力**:  核心功能模块（2-3 句话）



### 2.用户角色与场景

#### 包含:

#### 2.1用户角色

| 角色  | 角色诉求                        |
| ----- | ------------------------------- |
| User1 | Read...                         |
| User2 | Create..., Update..., Delete... |
| User3 | Drag...                         |

#### 2.2核心场景

| 场景    | 触发             | 目标结果    |
| ------- | ---------------- | ----------- |
| case1： | 点击...          | 产生结果... |
| case2： | 双击...          | 产生结果... |
| case3： | 拖拽...          | 产生结果... |
| case4： | 输入...，回车... | 产生结果... |



### 3.范围边界

#### 3.1 包含在本次范围内

- **功能点 1：** 功能描述
- **功能点 2：** 功能描述
- **功能点 3：** 功能描述

#### 3.2 超出本次范围

- 明确规定我们不做的事情
- 未来迭代的考量因素



### 4.业务规则与业务模型结论(非必须)

#### 4.1核心业务流程

#### 4.2关键业务规则

#### 4.3状态与异常处理

#### 4.4业务数据口径



### 5.功能需求列表

| ID   | 需求描述         | 备注 |
| ---- | ---------------- | ---- |
| FR1  | User can...      |      |
| FR2  | System should... |      |
| FR3  | Feature must...  |      |

需求描述与备注的内容应该包含：

- 使用角色：
- 业务目的：
- 前置条件：
- 功能说明：
- 规则约束：
- 异常处理：
- 数据要求：
- 权限要求：
- 验收条件：





### 7.原型与交互说明

#### 7.1 设计原则

- **原则 1：** 原则描述
- **原则 2：** 原则描述
- **原则 3：** 原则描述

#### 7.2 页面结构与交互流程

- 核心页面结构及交互流程：左右分栏布局
  - 左侧：占页面宽度的...，展示...
  - 右侧：占页面宽度的...，展示...
- 分隔线：左右两栏

#### 7.3 信息架构

- 导航结构
- 数据组织逻辑
- 内容层级划分



### 8.非功能需求

- **性能：** 响应时间、吞吐量指标
- **可扩展性：** 用户增长及数据量增长的预期目标
- **安全性：** 身份验证、权限控制、数据保护
- **可靠性：** 系统在线率目标（Uptime）、容错与错误率
- **易用性：** 无障碍设计标准（Accessibility）、多设备适配支持
