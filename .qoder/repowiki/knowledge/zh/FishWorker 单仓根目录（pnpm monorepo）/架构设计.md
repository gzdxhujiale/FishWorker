根目录作为 pnpm monorepo 入口，将三个子模块按职责分层：`frontend_app`（React + Arco Design + Zustand + TipTap）通过 `@tauri-apps/api` 调用 Rust 命令；`tauri_backend`（Rust + Tauri v2）暴露持久化到 MySQL 的命令，供前端桥接；`docs` 存放 PRD/接口规范等 Markdown。

跨子模块的装配点集中在根级配置：
- `vite.config.ts` 固定开发端口 1420/HMR 1421，忽略 `src-tauri` 变更，使 Vite 仅负责前端热更新；
- `package.json` 的 `tauri:dev` / `tauri:build` 脚本由 `@tauri-apps/cli` 驱动，内部再启动 Vite 并加载 Rust 后端；
- `tsconfig.json` 通过 `paths` 与 `references` 把 `quill` 解析到本地 stub，同时引用 `tsconfig.node.json` 区分 Node 侧类型；
- `pnpm-workspace.yaml` 允许 `esbuild` 与 `@parcel/watcher` 在 Tauri 环境中编译原生插件。

前后端契约以 `tauri_backend` 暴露的 Rust 命令为边界，`frontend_app` 通过 Tauri 插件 API 单向调用，数据库连接与文件 I/O 被严格限制在后端进程内。
