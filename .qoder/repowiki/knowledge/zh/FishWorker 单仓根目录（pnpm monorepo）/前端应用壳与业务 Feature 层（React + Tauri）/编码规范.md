- 每个 Feature 目录包含 `*Store.ts`（Zustand create + init 异步加载）、`*Service.ts`（I/O 抽象）、`*Types.ts`（领域类型）三件套，UI 组件只 import store 与 service，不直接访问存储。
- 所有写操作先 optimistic 更新本地 Zustand state，再通过 `syncEngine.schedule(resourceKey, () => service.upsert(...), delay)` 去抖落盘，删除动作额外调用 `syncEngine.cancel(resourceKey)` 取消待持久化任务。
- 初始化统一通过 store 的 `init()` 方法完成：优先从 TiDB 加载，失败回退到 localStorage，并在首次启动时执行 `migrateFromLocal` 迁移旧数据。
- ID 生成统一使用 `genId(prefix)` 模式（`${prefix}-${Date.now()}-${random}`）或 `crypto.randomUUID()`，避免手写字符串拼接。
- 列表/分组排序逻辑抽离为纯函数（如 `listsReorder.ts` 中的 `computeNoteReorder`），由 dragEnd 事件计算 action 后交由 store 执行，保持 UI 与算法解耦。
- 组件样式采用 CSS Modules 风格：组件同名 `.css`/`.scss` 文件就近放置，SCSS 变量通过 `_variables.scss` 集中声明，避免在 JSX 中写内联样式（除临时占位外）。
