# 产品需求文档 (PRD) - 清单模块 v1.6

## 1. 概述
### 包含:
- **问题陈述：** 用户需要一个专门的模块来创建、分类和管理各种待办清单或笔记，以便于更好地组织个人任务和信息，提升个人时间管理和信息检索的效率。
- **建议的解决方案：** 新增“清单”功能页面，采用经典的左侧导航+右侧内容的页面结构，支持清单的快捷创建、个性化属性配置（如图标、颜色、视图等）以及通过文件夹进行层级化管理。引入快捷悬浮菜单、笔记抽屉式编辑和模板功能，提升内容记录体验。

## 2. 核心理念
通过灵活的清单分类、视觉化的高亮配置（颜色和图标）以及无缝的交互体验，帮助用户构建结构化、个性化的个人待办与知识管理体系。

## 3. 解决方案概述

### 3.1 建议的解决方案
- **高层级描述**: 构建一个全新的“清单”页面，包含左侧侧边栏（导航）和右侧主内容区。侧边栏用于展示所有清单及文件夹树形结构；右侧用于展示当前选中清单的详细内容。右侧内容区支持笔记的快速录入（输入框回车），并通过右侧滑出的抽屉面板进行详细编辑。
- **核心能力**: 清单的创建与管理、文件夹层级管理、清单属性自定义、快捷悬浮操作菜单、笔记的抽屉式编辑、笔记模板库。

### 3.2 包含在本次范围内
- **功能点 1：清单页面基础框架** - 实现左右分栏布局结构。
- **功能点 2：左侧导航栏展示及交互** - 清单列表展示、文件夹层级嵌套展示、hover 标题显示添加按钮。
- **功能点 3：侧边栏更多操作菜单** - 鼠标悬停文件夹或清单时显示“...”更多菜单，支持编辑、置顶、复制、删除等快捷操作，并规范了对应弹窗的一致性。
- **功能点 4：“添加/编辑清单”与“文件夹”弹窗** - 支持设置清单名称、图标、颜色、视图类型（列表/看板等），统一添加与编辑的 UI 样式。
- **功能点 5：内联文件夹管理** - 支持在创建清单时下拉选择已有文件夹，或直接在下拉菜单中新建文件夹。
- **功能点 6：笔记管理与抽屉编辑** - 清单内容区顶部提供笔记录入框（回车创建），笔记列表支持 Hover 操作菜单（置顶、移动到、复制、删除）。点击笔记可唤出右侧抽屉进行标题和内容的详细编辑。
- **功能点 7：笔记模板引导** - 清单为空时提供引导，完整显示“记录你的想法，或 使用模板”，支持点击弹出“笔记模板”弹窗选择并应用模板。
- **功能点 8：自定义确认与列表拖拽** - 采用自定义气泡确认框替代系统原生确认，清单支持拖拽排序。
- **功能点 9：清单内部分组管理** - 清单内容区支持创建分组，将笔记进行分类管理，支持分组折叠与安全删除。
- **功能点 10：笔记模板管理与应用优化** - 鼠标悬浮笔记模板时，右上角浮现编辑和删除按钮，支持管理模板库；应用模板时仅覆盖内容，保留原笔记标题。
- **功能点 11：拖拽交互视觉增强** - 在拖拽清单到文件夹或拖拽笔记到分组时，目标区域增加明显的视觉提示（如高亮背景），提升用户对放置位置的感知。

### 3.3 超出本次范围
- 复杂的富文本笔记编辑器功能（当前侧重于基础的标题/内容编辑和属性管理）。
- 多级嵌套文件夹（目前仅支持单层文件夹分类，即文件夹下直接是清单）。

## 4. 用户故事与需求

### 4.1 用户故事
```
作为一名 需要管理多项工作和生活事务的用户
我希望 能够在侧边栏快捷编辑或置顶我常用的清单和文件夹
以便于 我能更高效地访问最重要的内容

作为一名 经常记录会议或周报的用户
我希望 在新建笔记时能直接使用模板
以便于 统一格式，节省排版时间
```

### 4.3 非功能需求
- **易用性 (Usability)：** 侧边栏 Hover 菜单响应需迅速；右侧抽屉滑出动画需流畅平滑，抽屉外点击关闭符合用户直觉。
- **UI/UX：** 需高度还原设计稿，保持添加/编辑弹窗（清单、文件夹）的样式一致性；空白引导页中文案的点击区域需明显（如添加下划线或高亮颜色）。

## 5. 设计与用户体验

### 5.1 设计原则
- **上下文内操作：** 尽可能在用户光标悬停的地方提供相关操作（Hover 菜单），减少鼠标移动距离。
- **平滑过渡：** 笔记编辑采用抽屉模式而非页面跳转，帮助用户保持在当前清单上下文中的感知。
- **统一性：** “添加”与“编辑”的弹窗组件应尽可能复用，保证用户体验的一致性。

### 5.2 页面结构与交互流程
- **页面布局：** 
  - **左侧边栏**：清单导航，支持清单拖拽排序。
  - **中/右侧内容区**：顶部为笔记快捷输入框，下方为清单内的笔记列表（按分组展示，默认为无分组/未分组）。
  - **右侧抽屉 (Drawer)**：点击笔记时浮现于最右侧或覆盖内容区右半部，用于笔记编辑。点击其外部遮罩层关闭。
- **交互流程（更新）：**
  1. 笔记移动与排序：支持直接拖拽笔记以调整顺序或将其移入其他分组。也可通过 Hover 菜单将笔记跨清单转移。
  2. 分组管理：未分组不可改名或删除。分组标题最右侧显示笔记数，Hover 时变为下拉菜单。删除分组时无需确认，笔记自动归入“未分组”。
  3. 清单拖拽：清单支持在侧边栏中自由拖拽排序，支持拖出或拖入文件夹。

### 5.3 信息架构
- 侧边栏
  - 文件夹 (Hover: 添加清单(带默认文件夹)/编辑文件夹/置顶)
    - 清单 (右侧显示笔记数, Hover 变为菜单: 编辑/置顶/复制/删除, 支持拖拽进出文件夹)
- 主内容区
  - 顶部操作区: [新建笔记输入框...] / 更多操作“...” (创建分组)
  - 分组 A (最右侧显示笔记数, Hover 变为菜单: 重命名/删除, 可折叠)
    - 笔记项 1 (支持拖拽排序和改组, Hover: 置顶/移动到/创建副本/删除)
  - [未分组] (不可重命名/删除)
    - 笔记项 2

## 6. 技术规格说明建议

### 6.1 前端组件划分
- `ListLayout`, `Sidebar`, `AddListModal`, `FolderModal`
- **新增/更新组件**:
  - `HoverContextMenu`: 通用的 Hover 更多操作下拉菜单组件。
  - `ConfirmBubble`: 悬浮确认气泡框组件，用于替换 `window.confirm`。
  - `NoteGroup`: 清单内容区的分组容器组件。
  - `NoteItem`: 独立的笔记项组件，包含 Hover 时的下拉菜单和移动至搜索面板。
  - `NoteList`: 清单主内容区的笔记列表组件。
  - `NoteDrawer`: 右侧滑出的抽屉组件。

### 6.2 数据模型定义 (参考)
- **List Entity**: 
  - `id`: string, `name`: string, `folderId`: string | null, `isPinned`: boolean
  - `sortOrder`: number (连续型整数，用于拖拽排序)
- **NoteGroup Entity** (新增):
  - `id`: string
  - `listId`: string
  - `name`: string
  - `sortOrder`: number
- **Note Entity**:
  - `id`: string
  - `listId`: string
  - `groupId`: string | null (归属的分组，null 为未分组)
  - `sortOrder`: number (连续型整数，用于拖拽排序)
  - `title`: string
  - `content`: string
  - `isPinned`: boolean
  - `createdAt`, `updatedAt`: timestamp
- **Template Entity**:
  - `id`: string
  - `name`: string
  - `content`: string
