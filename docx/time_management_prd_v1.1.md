# PRD: 高效能时间管理模块 v1.1 (基于《七个习惯》)

## 1. 概述 (Overview)
本模块旨在将《高效能人士的七个习惯》中的“要事第一”原则与“四象限时间管理法”结合，帮助用户跳出日常琐事，聚焦于真正重要的事情。通过“以角色为中心的周计划”和“以四象限为基础的日执行”，实现从宏观目标到微观行动的落地。

**v1.1 迭代目标：**
在核心管理流程的基础上，丰富任务的表达能力（截止时间、详细内容），并优化页面交互体验（隐藏已完成任务、支持任务删除），使任务管理更加灵活和清晰。

## 2. 核心理念 (Core Concepts)
*   **原则驱动**: 以价值观和原则为核心，而不是被突发事件牵着鼻子走。
*   **角色与目标 (Roles & Goals)**: 每个人在生活中扮演不同的角色（如：开发者、父母、自我提升者等）。每周为每个角色设定 1-2 个真正重要（第二象限）的目标。
*   **四象限法则 (Eisenhower Matrix)**:
    *   **Q1: 重要且紧急** (危机、急迫的问题、有期限的任务) - *必须立刻做，但过多会导致压力疲劳*
    *   **Q2: 重要但不紧急** (准备、预防、价值观澄清、计划、关系建立、真正的休闲) - ***高效能的核心聚焦点***
    *   **Q3: 紧急但不重要** (干扰、一些电话/会议、凑热闹的事情) - *尽量授权或拒绝*
    *   **Q4: 不重要且不紧急** (琐事、耗费时间的消遣、无关紧要的信件) - *尽量消除*

## 3. 页面设计 (Page Design)

本模块主要由两个核心页面构成：**「周计划与角色」** 和 **「四象限工作台」**。

### 页面一：周计划与角色 (Weekly Planning)
**设计目标**：在每周开始前，引导用户思考自己的各个角色，并为每个角色设定本周的“要事（Q2目标）”，然后将其安排到具体日程中。

**主要功能模块**：
1.  **角色管理与目标池 (Role & Goal Backlog)**
    *   **角色卡片**：支持添加、编辑、删除角色（例如：个人成长、丈夫/妻子、前端开发等）。支持为角色配置 Icon 或主题色。
    *   **周目标录入**：在每个角色卡片内，可以快速添加本周需要达成的目标/任务（Input + Enter）。
    *   **[v1.1 新增] 任务删除功能**：在角色与目标池中，支持直接删除任务/目标，方便用户清理不再需要或误建的任务。
    *   **默认属性**：此处录入的目标，在概念上默认属于“第二象限（重要不紧急）”的要事。
2.  **周日程看板 (Weekly Schedule Board - Kanban Style)**
    *   **视图布局**：展示周一至周日共 7 个列的看板（或周历视图）。
    *   **拖拽规划 (Drag & Drop)**：支持将左侧“目标池”中的任务拖拽到具体的某一天中进行排期。
    *   **[v1.1 新增] 已排期任务删除**：在周日程看板中，支持直接删除任务，撤销不合理的规划安排。
    *   **负荷提示**：显示某一天已分配的任务数量，防止单日规划过度。
    *   **[v1.1 新增] 隐藏已完成任务**：提供“隐藏已完成”的开关，开启后，看板中已勾选完成的任务将不再显示，保持页面清爽。

### 页面二：四象限工作台 (Daily Quadrant Execution)
**设计目标**：作为每日的主工作台，管理当天的具体任务，并在四象限矩阵中进行分类和执行，确保每天的精力集中在正确的事情上。

**主要功能模块**：
1.  **全景四象限视图 (Four Quadrants Matrix)**
    *   **视图布局**：屏幕主体划分为 2x2 的四个象限区域（Q1, Q2, Q3, Q4）。
    *   **快速添加**：每个象限区域内都支持快速输入新增任务。
    *   **拖拽流转**：支持在四个象限之间自由拖拽任务以改变其重要/紧急属性。
    *   **[v1.1 新增] 隐藏已完成任务**：在工作台顶部提供全局“隐藏已完成”开关。开启后，各象限中已完成的任务将被隐藏，减少视觉干扰，让用户专注当前未完成的要事。
    *   **视觉规范**：
        *   Q1 (重要且紧急)：红色基调，强调紧迫感。
        *   Q2 (重要不紧急)：绿色/蓝色基调，强调清晰、平和与专注。
        *   Q3 (紧急不重要)：黄色/橙色基调，提醒谨慎对待。
        *   Q4 (不重要不紧急)：灰色基调，视觉弱化。
2.  **任务详情管理 [v1.1 重点强化]**
    *   任务除了基础的“标题”外，新增以下扩展属性：
        *   **[v1.1 新增] 详细任务内容 (Description)**：支持录入多行文本作为任务的补充说明、备注或执行步骤。
        *   **[v1.1 新增] 截止时间 (Deadline/Due Date)**：支持为任务设定具体的截止时间（日期+具体时间）。当任务临近或超过截止时间时，在 UI 上给予高亮或醒目的视觉提示（如：变红、显示“已逾期”标签）。
3.  **数据联动与“今日聚焦” (Today's Integration)**
    *   如果在「周计划」页面中，有被分配到“今天”的任务，这些任务在用户打开工作台时，会**自动同步**并默认出现在 **Q2（重要不紧急）** 象限中。
    *   如果到了截止日期（例如昨天规划的任务未完成），可以根据设置自动升级流转到 Q1（重要且紧急）。
4.  **任务操作与执行 (Task Execution)**
    *   **完成状态**: 点击复选框完成任务，任务文字加上删除线并下沉到底部或归档（如果未开启隐藏）。
    *   **专注模式 (可选进阶)**: 针对 Q2 或 Q1 中的重点任务，提供“启动番茄钟”或专注计时的入口。
    *   **一键延期**: 对当天未完成的任务，提供快速推迟到明天，或退回到“周计划目标池”的操作。

## 4. 典型用户旅程 (User Journey - v1.1 版)
1.  **周末规划 (周计划页面)**：周末或周一早晨，用户打开「周计划」页面。为“开发工作”定下重构某模块的目标。在添加目标时，用户**详细补充了重构的思路和注意点（详细任务内容）**，并将这个目标拖拽到周二。同时，发现目标池中有个冗余的任务，**直接点击删除按钮进行了清理（任务删除）**。
2.  **每日执行 (四象限工作台)**：周二早晨，用户打开「四象限工作台」，“重构模块”已经在 Q2 中。用户为该任务**设置了今天下午 18:00 的截止时间（截止时间）**。
3.  **清爽专注**：完成一项小任务后，用户觉得页面上的已完成任务有点多，点击了**“隐藏已完成任务”开关（隐藏功能）**，界面瞬间清爽，用户得以集中精力在未完成的重构任务上。

## 5. 数据模型更新参考 (Data Model Updates)
为了支持 v1.1 的新功能，`Task` 数据模型需要做相应的扩展：
```typescript
export interface Task {
  id: string;
  title: string;
  roleId?: string; 
  quadrant: QuadrantType;
  scheduledDate?: string; 
  completed: boolean;
  createdAt: number;
  completedAt?: number;
  // v1.1 新增字段
  description?: string; // 任务详细内容
  deadline?: number;    // 截止时间 (时间戳或日期字符串)
}
```

## 6. 后续迭代方向 (Future Iterations)
*   **精力统计 (Analytics)**: 统计用户每周在四个象限完成任务的数量和比例，引导用户逐步实现“Q2 占比最大化，消灭 Q4”的理想状态。
*   **长期愿景板 (Vision Board)**: 在角色之上增加使命宣言 (Mission Statement) 或年度目标，使得周目标有更清晰的源头。
*   **子任务 (Subtasks)**: 针对复杂的 Q2 任务，支持拆分为多个子任务列表。
