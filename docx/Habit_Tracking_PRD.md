# PRD: 习惯追踪模块 v1.0

## 1. 产品概述 (Overview)

### 背景与问题陈述
用户希望培养良好习惯并通过可视化方式追踪完成情况，但缺乏简单直观的工具来管理日常习惯。现有的待办清单无法有效支持习惯性行为的长期记录和进度展示。

### 产品目标
- 提供一个简洁的习惯追踪界面，帮助用户建立和坚持日常习惯
- 通过打卡机制、连续天数统计、完成率等指标激励用户持续行动
- 以周视图和日历热力图的形式直观展示习惯执行情况

### 高层级描述
习惯追踪模块是一个独立的 AppSection，集成到 FishWorker 主应用中。页面采用上下布局（3:7 比例），上半部分为日期切换器，下半部分为当天习惯列表。点击习惯可展开侧边栏查看详细数据，包括概览卡片、日历热力图等。

### 核心能力
- 新增/编辑/删除习惯的基本 CRUD 操作
- 按日期查看习惯列表并打卡完成
- 展示累计坚持天数和当前连续天数
- 显示月度打卡统计和完成率
- 日历热力图可视化当月执行状况

## 2. 用户角色与场景

### 2.1 用户角色

| 角色 | 角色诉求 |
|------|---------|
| User1 | 创建新习惯，设置习惯名称和基本属性 |
| User2 | 每天打卡已完成的习惯，跟踪进度 |
| User3 | 查看历史数据和统计数据，分析习惯养成情况 |
| User4 | 管理习惯列表，编辑或删除不再需要的习惯 |

### 2.2 核心场景

| 场景 | 触发 | 目标结果 |
|------|------|---------|
| case1：创建习惯 | 点击右上角"新建习惯"按钮 | 弹出表单填写习惯名称，保存后出现在习惯列表中 |
| case2：打卡习惯 | 点击习惯右侧的圆点开关 | 圆点变绿，该习惯当天的打卡状态被记录 |
| case3：查看详情 | 点击习惯列表中的任意习惯项 | 从右侧滑出侧边栏，展示习惯的详细统计和日历热力图，点击侧边栏其他区域可关闭侧边栏。 |

| case6：切换日期 | 点击上半部分的日期导航箭头 | 切换至前一天或后一天，显示对应日期的习惯列表 |
| case7：空状态查看 | 切换到没有安排任何习惯的日期 | 显示提示「你这一天没有习惯」 |

## 3. 范围边界

### 3.1 包含在本次范围内

#### 功能点 1：习惯 CRUD 操作
- **描述**：支持新建、编辑、删除习惯
- **交互要求**：
  - 点击"新建习惯"按钮弹出模态框，输入习惯名称即可创建
  - 在习惯详情侧边栏右上角提供下拉菜单，可选择编辑或删除
  - 删除前需二次确认防止误操作
  - 编辑时可修改习惯名称

#### 功能点 2：日期切换与习惯列表展示
- **描述**：页面分为上下两部分，上半部分日期切换，下半部分习惯列表
- **布局要求**：
  - 上半部分占比 3，展示日期选择器（周一到周日）
  - 下半部分占比 7，展示选中日期的习惯列表
  - 初始化时自动选择当天日期
  - 未选择日期时高亮显示

#### 功能点 3：打卡机制
- **描述**：用户可以通过点击圆点标记习惯完成情况
- **交互要求**：
  - 每个习惯项右侧有一个圆形开关
  - 未打卡状态下圆点为空心灰色，点击后变为实心绿色
  - 只允许对当天及未来的日期进行打卡操作
  - 过去的习惯打卡选项隐藏不可见
  - 点击操作立即生效，无感保存

#### 功能点 4：统计数据展示
- **描述**：每个习惯显示关键统计数据
- **展示内容**：
  - 总坚持天数：图标 + 数字形式展示累计打卡次数
  - 当前连续天数：图标 + 数字形式展示连续打卡 streak
  - 数据实时同步到详情页和侧边栏

#### 功能点 5：详情侧边栏
- **描述**：点击习惯项滑出侧边栏展示详细信息
- **布局要求**：整体垂直布局，比例为标题栏:概览卡片：日历热力图 = 1:3:6
- **组成部分**：
  - **标题栏**：习惯名称 + 三点下拉菜单（编辑/删除）
  - **概览卡片**：2×2 网格布局，展示 4 个统计卡片
    - 月打卡数（本月打卡次数）
    - 总打卡数（累计打卡次数）
    - 月完成率（本月完成百分比）
    - 当前连续天数（streak 计数）
  - **日历热力图**：展示当月每日打卡状态的热力图
    - 已打卡日期显示深绿色
    - 未打卡日期显示浅灰色

#### 功能点 6：空状态处理
- **描述**：当所选日期没有安排任何习惯时的提示
- **文案要求**：显示「你这一天没有习惯」
- **建议操作**：提供快速跳转到习惯创建入口的引导

### 3.2 超出本次范围

## 4. 业务规则与业务模型

### 4.1 核心业务流程

```
用户进入习惯页面
  ↓
自动加载当天日期
  ↓
显示当天习惯列表（如有）
  ↓
用户可选择：
  ├─ 点击圆点打卡 → 更新打卡状态 → 实时更新统计数据
  ├─ 点击习惯项 → 滑出详情侧边栏 → 查看详细数据
  │                     └─ 可进行编辑/删除操作
  └─ 切换日期 → 加载对应日期习惯列表
```

### 4.2 关键业务规则

| 规则 ID | 规则描述 |
|--------|---------|
| BR1 | 默认日期初始化为当天，并高亮显示 |
| BR2 | 仅能对自己及未来日期的习惯执行打卡操作 |
| BR3 | 历史日期的打卡圆点隐藏，不可点击 |
| BR4 | 打卡状态变更立即反映到统计数据中（乐观更新） |
| BR5 | 删除习惯前必须弹出确认对话框 |
| BR6 | 侧边栏点击遮罩层或关闭按钮时关闭 |
| BR7 | 概览卡片 4 个数据项始终同时展示，2×2 等分布局 |

### 4.3 习惯每日展示与过滤规则 (新增)

根据自定义的“开始日期”、“频率”和“坚持天数”，系统需动态计算某一天是否展示某个习惯：

1. **基于「开始日期 (startDate)」的过滤**：
   - 习惯不应出现在其 `startDate` 之前的日期列表中。
   - 如果未设置 `startDate`，则默认使用习惯的创建日期 (`createdAt`) 作为开始日期。
2. **基于「坚持天数 (duration)」的过滤**：
   - 若 `duration` 为 `forever`（永远），则该习惯在开始日期之后的每一天都可能展示，无结束限制。
   - 若 `duration` 为 `21days`（21天）或其他具体天数，则需计算出结束日期（即 `startDate` + `duration` 天数）。在该结束日期之后，习惯将不再出现在列表中。
3. **基于「频率 (frequency)」的过滤/展示**：
   - 若 `frequency` 为 `everyday`（每天），在有效的日期范围内（开始日期到结束日期），每一天都会展示该习惯。
   - 若 `frequency` 为 `weekly`（每周），该习惯会在有效日期的每一天展示。若用户在本周内（周一至周日）的任意一天完成了打卡，其余日期的列表可保持展示，但视觉上建议弱化（例如：无需重复打卡，或复用当周已打卡状态）。

### 4.4 状态与异常处理

| 场景 | 处理方式 |
|------|---------|
| 网络连接中断 | 本地缓存打卡操作，网络恢复后同步；失败时给出提示 |
| 习惯为空 | 显示空状态页面，文案「你这一天没有习惯」 |
| 删除习惯有历史记录 | 确认提示中包含「将同时删除所有打卡记录」说明 |
| 侧边栏打开时切换日期 | 自动关闭侧边栏，避免数据不一致 |

### 4.5 业务数据口径

| 指标 | 计算方式 |
|------|---------|
| 月打卡 | 本月内已打卡的次数 |
| 总打卡 | 所有历史已打卡次数累加 |
| 月完成率 | 月打卡数 ÷ 本月应有打卡天数 × 100% |
| 当前连续 | 从昨天往前倒推连续打卡的天数，中断则归零 |

## 6. 原型与交互说明

### 6.1 设计原则

- **极简优先**：减少非必要元素，专注于核心打卡行为
- **即时反馈**：所有操作立即在 UI 上响应，无需手动保存
- **视觉引导**：通过颜色（绿/灰）直观区分打卡状态
- **渐进披露**：详细信息隐藏在侧边栏中，降低主页面复杂度
- **手势友好**：大尺寸点击区域，适配鼠标和触摸操作

### 7.3 数据统计逻辑

#### 月打卡次数
```typescript
getMonthCheckIns(year: number, month: number) {
  return this.checkIns.filter(ci => 
    ci.habitId === habitId && 
    new Date(ci.date).getFullYear() === year && 
    new Date(ci.date).getMonth() === month &&
    ci.completed
  ).length;
}
```

#### 月完成率
```typescript
getMonthlyCompletionRate(habitId: string, year: number, month: number) {
  const totalDays = daysInMonth(year, month);
  const checkedDays = this.getMonthCheckIns(year, month);
  return Math.round((checkedDays / totalDays) * 100);
}
```

#### 当前连续天数
```typescript
getCurrentStreak(habitId: string) {
  let streak = 0;
  const today = new Date();
  
  for (let i = 0; i < 365; i++) {
    const checkDate = subDays(today, i);
    const hasCheckedIn = this.hasCheckedIn(habitId, checkDate);
    
    if (hasCheckedIn || i === 0) {
      if (hasCheckedIn) streak++;
    } else {
      break;
    }
  }
  
  return streak;
}
```

### 7.5 Tauri 后端集成

数据库表结构设计：

```sql
-- 习惯表
CREATE TABLE habits (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 打卡记录表
CREATE TABLE habit_checkins (
  id TEXT PRIMARY KEY,
  habit_id TEXT NOT NULL,
  checkin_date DATE NOT NULL,
  completed BOOLEAN DEFAULT TRUE,
  FOREIGN KEY (habit_id) REFERENCES habits(id),
  UNIQUE(habit_id, checkin_date)
);

-- 索引优化
CREATE INDEX idx_checkins_habit_date ON habit_checkins(habit_id, checkin_date);
```
