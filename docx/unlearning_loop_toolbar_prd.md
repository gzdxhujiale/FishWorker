# PRD - 反学习回路（Unlearning Loop）工具栏

## 1. 概述 (Executive Summary)

**目的**: 为用户提供一个结构化、可视化的工具栏，支持设计和实践“输入-输出-反馈-反思”的反学习（Unlearning）回路，帮助用户打破固有思维模式，重塑认知。

### 核心组件:
- **问题陈述**: 在复杂多变的工作与学习环境中，用户容易陷入路径依赖和思维定势，难以主动识别并摒弃过时或错误的知识与习惯。
- **解决方案**: 引入一个专用的“反学习”工具栏，引导用户通过标准的四步回路（输入新视角 -> 尝试新输出 -> 获取并记录反馈 -> 深度反思）来完成认知解构和重构。
- **业务影响**: 增强产品的差异化竞争力（不仅是笔记/知识管理，更是认知进化工具），提升用户的深度思考能力和长期留存率。
- **成功指标**: 工具栏唤起率、完整反学习回路的完成率、用户自评的认知提升满意度。

---

## 2. 问题定义 (Problem Definition)

### 2.1 用户问题
- **目标用户**: 知识工作者、学生、创意人员、研发人员。
- **具体问题**: 难以发现自己思维中的盲点；即使发现，也往往因为缺乏结构化的行动脚手架（Scaffolding）而停留在“想改变”的阶段，最终退回舒适区。
- **触发场景**: 在遇到创作瓶颈、复盘失败项目、或接触到具有颠覆性的新理论时。
- **深层原因**: 改变旧习惯和旧认知不仅需要动机，还需要对抗大脑的惰性，这需要明确的步骤指引。
- **负面影响**: 缺乏自我更新能力，限制了创新突破和个人的长远成长。

---

## 3. 解决方案概述 (Solution Overview)

### 3.1 提出的解决方案
- **高层描述**: 一个可悬浮或固定在编辑器/工作台边缘的“反学习”工具栏，包含四个状态节点：【输入 (Input)】【输出 (Output)】【反馈 (Feedback)】【反思 (Reflection)】。
- **用户旅程**:
  1. **输入阶段 (Input)**: 用户通过工具栏明确记录当前试图挑战的“旧认知”，并填入引入的“新视角/新信息”。
  2. **输出阶段 (Output)**: 工具栏引导用户基于新视角进行一次小规模的实践（如写一段不同风格的代码、换一个角度写大纲）。支持快速关联当前正在编辑的文档。
  3. **反馈阶段 (Feedback)**: 用户通过小规模测试获取反馈，并在工具栏面板中客观记录测试结果和外界评价。
  4. **反思阶段 (Reflection)**: 用户对新旧认知进行对比复盘，确认是否成功完成了“反学习”，提取新的原则。

### 3.2 包含范围内 (In Scope)
- 四阶段节点状态的可视化进度指示器（如进度条或点亮图标）。
- 针对每个阶段的专属浮窗/侧边栏输入表单与启发式提示（Placeholder/Prompt）。
- 快速绑定功能：将当前工作区的高亮文本或整个文档一键关联至某个阶段的记录中。
- 历史反学习回路的归档、列表查看与回顾功能。

### 3.3 不在范围内 (Out of Scope)
- 复杂的社交化反馈机制（如一键发送给好友/同事求评价，MVP版本目前仅支持用户自行记录）。
- AI自动抓取内容填充（MVP版本侧重用户主动思考，暂不引入AI代写，但可考虑AI作为提问者辅助反思）。

## 4. 用户故事与需求 (User Stories & Requirements)

### 4.1 用户故事
- `US-01`: 作为一名创作者，我希望能够明确地记录我要打破的旧观点，以便于我有一个清晰的靶子（输入阶段）。
- `US-02`: 作为一名学习者，我希望系统能强制引导我经历“输出”和“反馈”，因为我经常只停留在“输入新知识”的假象中。
- `US-03`: 作为一名知识管理重度用户，我希望能够回顾我过去的“反学习”历程，以便看到自己的认知成长轨迹。

### 4.2 功能需求 (Functional Requirements)
| 编号 | 需求描述 | 优先级 | 备注 |
|---|---|---|---|
| FR1 | 工具栏四步状态展示，点击可展开对应的交互面板 | P0 | 核心交互框架 |
| FR2 | **输入面板**：支持填写“当前旧认知”和“引入的新视角” | P0 | 提供字段校验，必填 |
| FR3 | **输出面板**：支持关联当前文档，或提供空白框用于新建实验草稿 | P1 | 强化行动力 |
| FR4 | **反馈面板**：提供“预期结果”与“实际结果/外界评价”的对比记录框 | P0 | 强调客观性 |
| FR5 | **反思面板**：提供总结框，完成后生成一张精美的“反学习卡片” | P0 | 形成闭环，增强成就感 |
| FR6 | 历史回路列表及状态筛选（进行中/已完成） | P1 | 方便持续追踪 |

## 5. 设计与用户体验 (Design & User Experience)

### 5.1 设计原则
- **轻量且无干扰**: 工具栏在平时应尽量收起或保持视觉上的克制（如半透明悬浮），绝对不打断用户的核心创作流。
- **引导性强**: 步骤之间应有清晰的逻辑指引，每一步的输入框内必须提供占位提示词（Placeholder）来启发用户思考（例如：“你过去总是认为……”）。
- **正向反馈机制**: 当用户推进阶段，特别是完成一个完整的反学习回路时，给予显著的视觉正向反馈（如微动效、徽章、生成可分享的卡片）。

---

## 6. 技术规范概览 (Technical Specifications)

- **组件架构**: 工具栏需设计为全局挂载的独立组件，具备自己的全局状态管理，以便于跨页面或跨文档时能够持续追踪同一个“反学习回路”的进度。
- **核心数据模型 (`UnlearningLoop`)**:
  - `id`: 唯一标识符
  - `status`: 枚举值 (INPUT, OUTPUT, FEEDBACK, REFLECTION, COMPLETED)
  - `input_data`: { old_belief: string, new_perspective: string }
  - `output_data`: { action_taken: string, linked_doc_id: string }
  - `feedback_data`: { expected: string, actual: string }
  - `reflection_data`: { key_takeaway: string }
  - `timestamps`: created_at, updated_at, completed_at

## 7. 成功指标 (Success Metrics)

| 核心指标 | 定义 | 目标值 (参考) |
|--------|---------|--------|
| **采用率 (Adoption)** | 每周至少新建或更新一次反学习回路的活跃用户占比 | > 10% |
| **完成率 (Completion)** | 新建的回路中，在30天内走完全部四个阶段的比例 | > 30% |
| **用户满意度 (CSAT)** | 用户对“该工具帮助我打破了思维定势”的评分 | > 4.2 / 5.0 |

## 9. 页面结构、UI/UX交互与视觉逻辑细节

### 9.1 整体架构与视觉风格 (参考现有实现)
- **面板形态**: 继承本项目中已有的 `AiAssistantPanel` 或 `TimeManagementPanel` 的布局模式，采用右侧侧边栏（Sidebar）或浮动面板（Floating Panel），确保操作流畅且不遮挡左侧的主内容区（如思维导图或文本编辑器）。
- **视觉语言**: 
  - 维持 `styles.css` 与 `index.css` 中定义的一致性：面板使用半透明毛玻璃背景（backdrop-filter），以及深浅色模式（Dark/Light Mode）自适应的背景色和圆角卡片（border-radius）。
  - 所有图标必须使用系统现有的 `lucide-react` 组件，保持统一的线条粗细与风格。
- **全局进度导航器 (Progress Indicator)**:
  - 面板顶部常驻一个由四个 Lucide 图标组成的横向进度指示器，代表四步节点：如 `Lightbulb` (输入) ➔ `Keyboard` (输出) ➔ `RefreshCw` (反馈) ➔ `CheckCircle2` (反思)。
  - 当前激活步骤以系统主色（Primary Color）高亮显示；已完成步骤显示 `Check` 图标状态；未到达步骤置灰并降低透明度。

### 9.2 四个核心面板 (The 4 Panels)

#### 【第一步】输入面板 (Input) - "打破旧知"
- **页面结构**:
  - **标题区**: "输入与挑战"
  - **字段一**: "当前旧认知" (Textarea)
  - **字段二**: "引入的新视角" (Textarea)
  - **操作区**: "进入下一步" 按钮（右侧带 `ChevronRight` 图标）。
- **UI交互与逻辑**:
  - 两个输入框初始占位符（Placeholder）设计为具有启发式的短句，例如 *"例如：我过去总是认为..."* 或 *"最近接触到的新观点指出..."*，以降低用户的思考启动成本。
  - Textarea 需支持高度自适应（Auto-resize），避免出现局促的内嵌滚动条。
  - **逻辑校验**：只有当两个字段均输入了有效文本时，“进入下一步”的主按钮才会解除 `disabled` 状态变为可点击的高亮状态。

#### 【第二步】输出面板 (Output) - "践行新知"
- **页面结构**:
  - **标题区**: "基于新视角的输出"
  - **创作选项卡**: 
    - **选项 A (绑定已有节点)**：提供一个树状选择器（可复用 `CourseSidebar` 或 `MindMapCatalog` 中的树结构 UI），允许用户选择当前主工作区（Word / Mindmap）中的某个特定节点作为本次输出的锚点。
    - **选项 B (新建快速草稿)**：在面板内嵌一个简易多行输入框，用于快速记录非正式的文字输出。
  - **操作区**: "进入下一步" 按钮。
- **UI交互与逻辑**:
  - 绑定成功后，UI 会生成一张“锚点卡片”，卡片左侧带有 `FileText` (文档) 或 `GitBranch` (导图节点) 图标。
  - **双向联动**：用户点击面板中的锚点卡片，左侧的主工作区视口将自动跳转、展开并高亮对应的文本段落或思维导图节点。

#### 【第三步】反馈面板 (Feedback) - "验证效果"
- **页面结构**:
  - **标题区**: "记录反馈与结果"
  - **输入对比区**: 
    - "预期结果 (Expected)" (Textarea)
    - "实际结果/外界评价 (Actual)" (Textarea)
  - **快捷标签 (Quick Tags)**: 在实际结果输入框的上方悬浮一排 Pill 形态的快捷标签（如：`与预期一致`, `部分偏离`, `完全颠覆`）。
  - **操作区**: "进入最终反思" 按钮。
- **UI交互与逻辑**:
  - 用户点击快捷标签时，对应的文本会自动带入并作为“实际结果”输入框的前缀，方便快速记录。
  - 面板顶部提供一个“详情折叠面板（Collapsible）”，点击可展开第一步填写的“新旧认知”，辅助用户在记录反馈时进行对照。

#### 【第四步】反思面板 (Reflection) - "完成闭环"
- **页面结构**:
  - **标题区**: "复盘与沉淀"
  - **总结字段**: "核心收获 (Key Takeaway)" (Textarea)
  - **底部操作组**:
    - 主操作："完成回路"（全宽按钮，带 `Check` 图标）。
    - 次操作："重试输出" (幽灵按钮/文字按钮，点击可退回至第二步重新尝试)。
- **UI交互与逻辑**:
  - **正向激励动效**：当用户提交并点击“完成回路”后，在面板中央触发短促的成功微动效（如打勾动画展开或轻度撒花），给予用户“完成一次思维进化”的心理成就感。
  - 数据随后被标记为 `COMPLETED` 状态，页面渲染出一张包含了四个阶段摘要的只读“反学习总结卡片”。
  - **导出与分享**：总结卡片底部提供操作栏，使用 `Download` 图标。用户点击后，可将整个回路的数据作为 Markdown 格式，一键追加到当前活跃文档的末尾，或保存到用户的“知识库”中。
