# 【时间管理-四象限工作台】产品需求文档 (PRD)

## 1. 概述
### 问题陈述
个人日程管理中常面临“清单过载”的问题。传统的线性待办列表无法直观反映任务的轻重缓急，导致用户极易优先处理易于完成的琐事（如Q4象限内的消遣与杂务）或被动应付紧急突发的打扰（如Q3象限的杂乱会议），从而挤占了真正具有核心价值的准备、规划与防范工作（Q2象限）。

### 建议的解决方案
本方案设计并实现了**「四象限工作台」**独立功能模块。在应用的主工具栏（Toolbar）中，**「四象限工作台」被拆分为一个独立的工具入口**，与「周计划」并列。工作台采用 2x2 的大屏矩阵视图，由于它是一级独立工具，因此天然不包含周计划的角色待办侧边栏，从而直接呈现极致沉浸的工作台环境。用户可通过拖拽直接在象限间流转任务，通过象限右上角的加号气泡窗实现场景化快速录入，并支持通过截止日期维度的折叠组对任务进行分类聚合。最关键的联动在于：**用户在「周计划」中拖动安排的任何周要事，都会自动同步并在「四象限工作台」的 Q2 列表中当天进行完美联动展示**。

---

## 2. 核心理念
- **艾森豪威尔法则 (Eisenhower Matrix)**：
  - **Q1 重要且紧急 (红色基调)**：危机、急迫的问题。需要立即执行。
  - **Q2 重要不紧急 (绿色基调)**：预防、规划、建立关系、学习提升。高效能人士的**核心聚焦区**，也是周计划任务落地的默认接收区。
  - **Q3 紧急不重要 (黄色/橙色基调)**：打扰、某些不必要的会议与琐碎应酬。尽量减少或授权他人。
  - **Q4 不重要不紧急 (灰色基调)**：消遣、时间浪费、垃圾邮件。应予以消除。
- **动态优先级调整**：任务的紧急程度会随着时间的推移（截止日期临近）发生演变，通过拖拽可在四个象限中自由调整其属性。

---

## 3. 解决方案概述

### 3.1 建议的解决方案
- **高层级描述**：
  「四象限工作台」是主工具栏中的独立功能项。页面独占全部内容区域，承载 2x2 矩阵网格。每个象限拥有独立的标题、快速添加气泡弹窗和任务折叠分组列表。
- **核心能力**：
  - **独立的一级工具入口**：与「周计划」在工具栏并列，提供无干扰的独立规划环境。
  - **周计划 Q2 自动映射（联动）**：接收来自「周计划」安排的排期要事（自动映射在当天的 Q2 象限中）。
  - **象限快捷添加入口**：每个象限右上角配备快捷添加按钮，弹出定位精确的轻量级 Popover。
  - **截止时间维度分类折叠**：象限内任务依据其 `deadline` 属性自动归入 5 个折叠组（一天内、三天内、一周内、一周外、无日期），并提供展开/收起能力。
  - **极简无把手拖拽**：任务卡片去除冗余边框、背景及 Grip 拖拽把手，仅保留底单线分隔，点击任意位置即可启动拖拽，实现象限间的快速划归。

### 3.2 包含在本次范围内
- **功能点 1：独立工具栏页面与 Q2 数据联动**
  - 在主导航中展示「四象限工作台」图标；进入该工具时，页面仅呈现四象限矩阵与全局设置。
  - 自动加载所有符合条件的周计划排期任务（`scheduledDate` 等于今日且 `quadrant` 为 `Q2` 且未完成）。
- **功能点 2：2x2 矩阵网格与主题色渲染**
  - 主渲染区域划分为 Q1-Q4 四个 Box，每个 Box 对应专有色调，包含顶部的圆形数字角标、象限名称以及典型案例提示。
- **功能点 3：快捷添加气泡框 (`QuickAddPopover`)**
  - 位于象限右上角，点击 "+" 唤出。
  - 气泡框内部包含任务标题输入框、日期选择器（选择截止日期）、标志以及提交/添加按钮。
  - 弹窗通过原生 DOM 边界计算，自动渲染在触发展示按钮的正下方（偏移量自适应）。
  - 支持点击外部自动关闭弹窗。
- **功能点 4：按截止时间折叠标签组 (`CollapsibleGroup`)**
  - 根据系统当前时间与任务 `deadline` 的差值自动聚合：
    - **一天内** (`diff <= 1` 天)
    - **三天内** (`diff <= 3` 天)
    - **一周内** (`diff <= 7` 天)
    - **一周外** (`diff > 7` 天)
    - **无日期** (`deadline` 未设置)
  - 分组头部显示该组内任务的总数，点击可展开或折叠。如果当前组任务数为 0，则自动隐藏，不渲染空标签。
- **功能点 5：极简任务项交互 (`tm-task-item-minimal`)**
  - 使用 `border-bottom` 代替复杂的卡片背景和边框，界面清爽。
  - 鼠标悬停时展示删除按钮，点击任务主体弹出“任务详情”模态框。
  - 任务项自身作为 Drag 源，拖拽进入其他象限的 Box 即可完成重要与紧急级别的调整。

### 3.3 超出本次范围
- 自动根据到期时间让任务强制从 Q2 自动升级到 Q1（目前由用户在每日规划中手动拖动流转，或通过排期系统统一分配）。
- 象限的高级统计图表与精力耗费记录。

## 4. 用户故事与需求

### 4.1 用户故事
```text
作为一名 [面临繁杂工作的项目经理]
我希望 [能在一个整洁、无干扰的 2x2 视图中快速查阅今天的任务，且直观看到哪些要在 1 天内截止]
以便于 [迅速决策先做什么、推迟什么，以及直接将目标拖入对应象限重订属性]

验收标准：
[x] 切换到四象限工作台，左侧边栏自动关闭，工作台布满屏幕。
[x] 点击 Q1 象限的 "+"，弹出快速输入气泡，输入标题并选定明天截止，点击添加即可归入该象限。
[x] 该任务被自动聚合在 Q1 看板下的【一天内】折叠列表内。
[x] 按住该任务拖入 Q2 看板中，任务背景随拖拽响应，松开后自动转移到 Q2 对应的折叠分类。
[x] 点击任务左侧的圆形 Checkbox，能即时切换完成状态并加上删除线。
```

### 4.2 功能需求
| ID | 需求描述 | 优先级 | 备注 |
|----|------------|---|-------|
| FR1 | 沉浸式视图侧边栏隐藏 | P0 | Tab 为 daily 时收起角色列表，展示大面积矩阵 |
| FR2 | 2x2 矩阵网格渲染 | P0 | 四个分区结构，视觉区分强烈 |
| FR3 | 象限内快速添加气泡 | P0 | 气泡表单：标题（必填）+ 截止日期（可选）+ 确认按钮，支持点击外部关闭 |
| FR4 | 截止时间动态分组 | P0 | 根据 `deadline` 精确到天的差值进行五级分类渲染 |
| FR5 | 分组折叠展开交互 | P1 | 头部交互触发收折，支持展示总数 |
| FR6 | 无把手全卡片拖拽 | P1 | 卡片任意位置按住即可拖拽，放置入目标象限更新 `quadrant` |
| FR7 | 极简线框样式与交互 | P1 | 去除复杂元素，支持 hover 显隐删除，支持点击弹窗编辑详情 |

### 4.3 非功能需求
- **交互性能**：拖放交互应极其流畅（使用原生 HTML5 Drag/Drop API）。
- **折叠状态保存**：折叠组件的展开/收起状态属于视图运行时状态，切换 Tab 后应默认保持展开。
- **自适应与防溢出**：各象限卡片标题过长时自动进行截断显示，保证 2x2 布局整齐不乱。

## 5. 设计与用户体验

### 5.1 设计原则
- **色彩暗示**：严格利用红色（Q1）、绿色（Q2）、黄色（Q3）、灰色（Q4）作为边界和组件角标的主色调，给用户直观的心理暗示。
- **信息分级**：任务以单行列表形式排列，去除卡片外框，依靠距离和淡实底线区分。利用分组来减少单屏大列表的信息压迫感。

## 6. 技术规格说明

### 6.1 前端核心组件列表
1. **`DailyQuadrants.tsx`**：
   - 核心视图入口，接受 `tasks: Task[]`，过滤属于 Q1-Q4 的任务。
   - 组织 2x2 网格，为各个象限分配截止日分类逻辑。
2. **`QuickAddPopover.tsx`**：
   - 使用 `position: 'fixed'` 绝对定位在触控点下方。
   - 提供 `autoFocus` 输入框和轻量日期控件。
3. **`CollapsibleGroup.tsx`**：
   - 支持子节点（Note/Task）的容器外包装。
   - 实现点击触发展开收折的局部 State。

### 6.2 状态管理与计算
在 `DailyQuadrants` 中，对每个象限的任务进行如下分类：
```typescript
const sortedTasks = [...qTasks].sort((a, b) => {
  if (a.completed === b.completed) return b.createdAt - a.createdAt; // 最新创建优先
  return a.completed ? 1 : -1; // 未完成任务排在最前
});

const now = Date.now();
const MS_PER_DAY = 24 * 60 * 60 * 1000;

sortedTasks.forEach(t => {
  if (!t.deadline) {
    noDate.push(t);
  } else {
    const diffDays = (t.deadline - now) / MS_PER_DAY;
    if (diffDays <= 1) within1Day.push(t);
    else if (diffDays <= 3) within3Days.push(t);
    else if (diffDays <= 7) within1Week.push(t);
    else beyond1Week.push(t);
  }
});
```
当拖放触发时，执行：
```typescript
onMoveTask(taskId, targetQuadrant); // 更新本地 store 及云端数据库，状态实时反馈给 UI
```
