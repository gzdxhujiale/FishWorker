- 所有对前端暴露的函数统一使用 `#[tauri::command]` 宏标注，并在 `lib.rs` 的 `generate_handler!` 中集中注册。
- 数据库操作全部采用 `sqlx::query(...).bind(...).execute/fetch_optional(&*pool).await` 风格，错误通过 `map_err(|e| e.to_string())?` 转为 String 返回。
- 长生命周期对象（MySqlPool）通过 `tauri::State<'_, MySqlPool>` 从命令参数注入，而非全局变量或闭包捕获。
- 配置读取遵循“多路径回退”模式：构造 PathBuf 向量后顺序遍历第一个存在的 JSON 文件，失败则返回默认结构体。
- 表结构变更集中在 `schema.rs::ensure_tables`，使用 `CREATE TABLE IF NOT EXISTS` 配合 `UNIQUE KEY` 保证幂等初始化。
- DTO 结构体统一使用 `#[serde(rename_all = "camelCase")]` 与前端字段名对齐，布尔列以 i8 存取并在映射时转换为 bool。
