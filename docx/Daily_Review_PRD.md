# 产品需求文档 (PRD) - 每日复盘模块 v1.0

## 1. 概述
### 包含:
- **问题陈述：** 用户需要记录每日的学习、反思及改进计划，以实现长期的习惯养成和能力复利增长。当前的编辑体验较为单一（普通 textarea 纯文本输入框），缺乏格式排版（加粗、列表、标题等）和流畅的富文本输入支持，限制了内容的表达与整理。此外，当前的高频输入同步机制在每次按键时直接保存，容易对后端数据库产生频繁写入的压力，需进一步优化性能。
- **建议的解决方案：** 为“每日复盘”模块集成 TipTap 富文本编辑器，支持 Markdown 语法快捷转换与常用富文本格式。在数据流动与保存上，引入“前端内存状态乐观更新 + 延迟防抖异步持久化”机制（打字时零延迟响应，停止输入 500ms 后自动异步保存至云端），并提供同步状态提示（同步中、已同步、同步失败），确保极致流畅且性能优异的编辑体验。

## 2. 核心理念
- **复利成长与习惯打卡 (Streaks & Compound Growth)**：通过连续记录天数（Streak）和数值化的“复利成长指标”（如 $1.01^{365} \approx 37.8$），给予用户视觉上的正向反馈与成就感，激励用户每日坚持复盘。
- **结构化深度记录 (Structured Reflection)**：通过 TipTap 编辑器，用户可以利用粗体、列表、引用块和任务列表，将反思内容组织得更为清晰，提升复盘质量。
- **零阻塞交互 (Optimistic & Fluid UI)**：任何编辑操作均在前端内存中瞬间完成更新，而向后端的网络/IPC 持久化由防抖定时器异步执行，绝不阻塞用户正常的键入流程。

## 3. 解决方案概述

### 3.1 建议的解决方案
- **高层级描述**：
  “每日复盘”页面分为左侧的复盘数据统计与日历打卡区，以及右侧的富文本复盘编辑区。整个页面拥有独立的每日打分、天数切换以及同步云端的状态展示。
- **核心能力**：
  - **成长与统计卡片**：展示当前复利值、总复盘天数、当前连续打卡天数和历史最高连续天数。
  - **打卡热力图**：按月份展示打卡记录，不同分数段（1-5星）对应不同的视觉高亮热力状态，支持点击日历日期快速加载对应日期的复盘。
  - **TipTap 富文本编辑器**：替换传统的 Textarea，支持 Markdown 语法解析（如 `# ` 变标题，`* ` 变无序列表等）、富文本修改与 placeholder 引导语。
  - **乐观更新与防抖保存**：打字时前端状态立刻更新，并在用户停止修改 500ms 后自动触发 Tauri 的 Rust API 将数据异步保存至云端。
  - **同步云端状态指示**：在编辑区域右上角实时指示当前数据的保存状态（“同步中...”、“已同步”、“同步失败”）。

### 3.2 包含在本次范围内
- **功能点 1：成长统计与打卡热力图 (CompoundStats)**
  - 根据连续记录天数自动计算复利值并高亮展示。
  - 渲染当月的打卡记录（使用 `react-day-picker`），每一天依据当天的评分（Rating：1-5星）显示不同程度的高亮徽标。
  - 支持通过日历日期选择、以及编辑区顶部的“前一天 / 后一天”按钮快捷流转日期，不可跨入未来日期。
- **功能点 2：TipTap 富文本编辑器集成 (ReviewEditor)**
  - 在 `ReviewEditor` 中使用 `@tiptap/react` 替换原有的 textarea 元素。
  - 支持配置 placeholder（例如：“今天学到了什么？什么可以改进？明天最重要的一件事？...”）。
  - 支持基本富文本操作（如粗体、斜体、列表、任务清单等）和 Markdown 快捷语法的自动渲染。
- **功能点 3：前端内存状态乐观更新与高频输入防抖 (Optimistic & Debounced Saving)**
  - 状态管理：编辑器的内容（content）发生变更时，前端优先更新内存中的 review 列表缓存，保持界面的响应速度。
  - 防抖计时器：当用户开始输入时，拦截立即持久化；当用户停止输入 `500ms` 后，再触发 `list_upsert_daily_review` 异步 IPC 指令。
  - 对于低频交互（如评分选择 1-5 星），不进行防抖，立即触发同步。
- **功能点 4：数据持久化**
  - 后端对接：每次防抖时间到达或点击星级评分时，调用 Tauri 命令，连接远端 TiDB 数据库将复盘记录进行 upsert。
- **功能点 5：无显式删除与自动判定删除**
  - 移除了显式的“删除复盘”按钮。
  - 当一天的复盘内容（TipTap 富文本 HTML）清空或为空时，判定为“当天没有复盘”。
  - 当用户清空编辑器内容，防抖 500ms 后自动触发后端 API 删除对应的复盘记录，同时更新本地状态，使得打卡日历中该日期的热力点熄灭、复盘统计数据及连续打卡天数重新计算。
- **功能点 6：菜单栏工具自适应动态流转机制 (Responsive Toolbar)**
  - 菜单栏/工具栏的富文本格式化工具随编辑器窗口/容器宽度的变化而动态调整展现方式。
  - 当编辑器宽度变小时，低频的高级格式化工具依次收起并隐藏至“更多格式”下拉菜单中；当编辑器宽度增加时，原本隐藏在下拉菜单内的功能按钮将自动“释放”回菜单栏展示，以确保在大屏/宽屏下功能直观可用，小屏/窄屏下界面简洁紧凑。
- **功能点 7：TipTap 拖拽组块与块操作下拉菜单 (Block Drag Handle & Dropdown Menu)**
  - 引入 TipTap 官方的 `@tiptap/extension-drag-handle`，支持鼠标悬浮到行/块左侧时显示拖拽柄（Drag Handle），允许拖拽组块进行顺序调整与排版。
  - **下拉菜单位置优化**：点击拖拽把手时，唤起的上下文菜单使用相对于视口的 `fixed` 定位，防止被复盘编辑区等具备滚动或裁剪特性的父容器裁剪隐藏。并且内置边界碰撞计算，当在视口边缘弹出时自动向上翻转（Flipping）或向左偏移，以确保菜单完整展现。
  - **相关功能的底层实现规格**：
    - **格式转换与结构转换**：通过 `setNodeSelection(pos)` 锁定被拖拽块的节点，然后再触发 `setParagraph`、`toggleHeading`、列表或引用转换命令。
    - **文字颜色与高亮设置**：通过 `setTextSelection(from, to)` 精准划定当前拖拽块内部文本的起止坐标，再应用 `setColor` 和 `toggleHighlight` Marks。
    - **块级复制/剪切/删除**：通过 `setNodeSelection(pos)` 对块进行整体选中，然后通过调用系统剪贴板指令或 `deleteSelection()` 准确作用于该块。
  - 点击该拖拽柄可弹出块操作下拉菜单，菜单结构如下：
    - **第一行（快捷文本/级别转换）**：正文（T 图标）、H1、H2、H3、H4、H5、H6，用于快捷将当前块转换为对应块级标签。
    - **第二行（内容结构与常用格式）**：有序列表（1/2列表图标）、无序列表、待办列表（打钩框图标）、引用（引号图标）、高亮/缩进对齐以及代码块（{}），支持快捷转换块结构或添加常用格式。
    - **下方常规列表选项**：
      - `段落颜色 >`（支持二级悬浮子菜单选择红、蓝、绿、黄等常用文字或背景高亮色）。
      - `缩进和对齐 >`（支持二级悬浮菜单控制段落对齐方式与缩进层级）。
      - 页面常规命令分割线。
      - `复制`（快速复制当前块内容）。
      - `剪切`（快速剪切当前块内容）。
      - `删除`（快速清空/删除当前选中的整个块级节点）。

### 3.3 超出本次范围
- 一次性导出所有日期的复盘记录生成长文档或 PDF 报告。
- 内置 AI 复盘助手（如根据内容自动分析当日收获并进行智能评级）。

## 4. 用户故事与需求

### 4.1 用户故事
```text
作为一名 寻求长期成长的自我管理者
我希望 在每天结束时在一个流畅、美观的编辑器里写下复盘内容
以便于 我可以随心所欲地使用列表和加粗排版，并且能在打字时不需要等待数据库保存的网络延迟，随时看到数据已成功同步的绿云标志。

验收标准：
[ ] 进入每日复盘页面，右上角显示“已同步”状态。
[ ] 在编辑器内快速键入字符时，编辑器响应完全流畅无延迟。右上角状态转为“同步中...”。
[ ] 停止打字 500ms 后，右上角重新变回绿色的“已同步”。
[ ] 用鼠标点击打分星级，评星立刻变化，且同步状态立即变为“同步中...”，保存完毕后恢复“已同步”。
```

### 4.2 功能需求
| ID | 需求描述 | 优先级 | 备注 |
|----|------------|---|-------|
| FR1 | 页面多面板展示 | P0 | 左侧展示复利成长指标与打卡日历热力图；右侧展示编辑器与评星区域 |
| FR2 | 日历评分热力点映射 | P0 | 绑定 `react-day-picker`，将各天 rating（1-5）划分为四种渐变绿/蓝背景点表示打卡频度与状态 |
| FR3 | 日期导航控制 | P0 | 支持“前一天/后一天”按钮切换，且“后一天”不能超过系统当前日期 |
| FR4 | TipTap 编辑器嵌入 | P0 | ReviewEditor 中载入 TipTap，具备 Markdown 语法实时转换与多格式支持 |
| FR5 | 内存状态乐观更新 | P0 | 用户键入内容时，直接保存至本地 state 中并反映在日历中，不发生卡顿 |
| FR6 | 编辑防抖持久化 | P0 | 用户编辑内容时，防抖防爆触发延迟设置为 `500ms`，停止编辑 500ms 后方将内容发送至 Tauri 后端进行云端持久化 |
| FR7 | 隐藏同步状态可视化指示 | P1 | 不提供“同步中...”/“已同步”/“同步失败”的视觉指示 |
| FR8 | 自动判定与内容级删除 | P0 | 移除显式“删除复盘”按钮。当复盘文本内容清空时，自动同步删除该记录，不计入总复盘天数及连续打卡统计。 |
| FR9 | 工具栏自适应动态流转 | P1 | 工具栏使用 `ResizeObserver` 动态监听容器宽度。在宽度受限时，逐步把部分格式化按钮（如对齐、上下标、图片等）收缩进“更多格式”下拉菜单中；大屏时自动释放到工具栏。 |
| FR10 | 拖拽组块与块级控制菜单 | P0 | 引入 TipTap 官方的 `DragHandle`。当鼠标悬停在块级节点时显示拖拽手柄。点击手柄弹出富功能块操作菜单。
- **锁定机制**：点击打开下拉菜单时，系统必须自动执行 `lockDragHandle()`，在菜单开启状态下锁定拖拽把手的位置，屏蔽其他块的 hover 漂移。菜单关闭时，执行 `unlockDragHandle()` 恢复锁定。
- **激活状态高亮**：菜单内各个选项的 active 状态必须直接与当前拖拽块 (`range.node`) 的类型、对齐及标记（颜色/高亮等属性）挂钩，禁止使用光标所在的 active 状态，保证展示的已有格式状态的真实性。
- **复制与剪切**：禁止使用 `execCommand`。复制与剪切均使用 `DOMSerializer.fromSchema` 对当前块进行 HTML 序列化，并通过 `navigator.clipboard.write` 写入富文本/纯文本，剪切动作完成后执行 `deleteSelection`。 |

### 4.3 非功能需求
- **交互流畅度**：打字时的响应延迟必须在 `16ms` 内（即 60FPS），不能因为后台网络写入或 IPC 同步导致编辑器输入卡顿或跳字。
- **防抖时效**：`500ms` 的防抖时间应精确，如果在防抖期内用户切换了复盘日期，需立刻对前一日期的修改执行“强制同步”，避免数据丢失。

## 5. 设计与用户体验

### 5.1 设计原则
- **极简而高级**：采用深色毛玻璃或柔和卡片布局，使用墨绿色/青色系列体现复利成长的“勃勃生机”。
- **状态不打扰**：同步状态图标仅在右上角以小字或云状小图标展示，确保用户能保持沉浸式写作“心流”。
- **自适应与动态释放**：界面上的交互组件应对不同窗口比例进行流体排布。高级选项“需要时出现”，低频工具按钮智能折叠，高频操作一触即达。

### 5.2 页面结构与交互流程
```text
+------------------------------------------------------------+
|  [菜单: 每日复盘]                                 ☁️ 已同步 |
+------------------------------------------------------------+
|  [左侧:成长卡片]               |  [右侧: 复盘编辑器]       |
|  * 复利成长值: 1.15x           |  <  2026-07-15 (今天)  >   |
|  * 总计复盘: 15天              |                            |
|  * 当前打卡/最高打卡: 8 / 12   |  [自适应工具栏: 动态流转]  |
|                                |  ========================  |
|  [左下: 日历热力图]            |  :: [ 拖拽柄 ] (悬浮展示)  |
|  [  7月  ]                     |  [ TipTap 富文本输入区 ]   |
|   1  2  3  4 (•) 5(•) 6        |  > 今天学到了什么？        |
|   7  8  9 10(•) 11    12       |  > 有什么地方可以改进？    |
|                                |                            |
|                                |  为今天状态打分: ★★★★☆     |
+------------------------------------------------------------+
```

#### 5.2.1 自适应工具栏动态流转细节
1. **大屏显示模式**（容器宽度 $\ge 720\text{px}$）：工具栏显示所有的文本格式化图标（包括：撤销/重做、标题下拉、粗体/斜体/下划线/删除线/高亮/链接、列表类型、缩进/对齐方式、图片上传）。
2. **小屏收缩模式**（容器宽度 $< 720\text{px}$）：工具栏的高级/低频操作（如：下划线、删除线、行内代码、引用、代码块、图片上传、对齐方式等）自动收拢进右侧的 `[更多格式]`（三点图标）下拉菜单内。
3. **流体流转表现**：在容器宽度从大到小变化的阶段，通过 CSS/JS 的 resize 触发状态更新，自动完成功能的“收纳”与“释放”，确保编辑区域不因工具栏过长折行而压缩高度。

#### 5.2.2 拖拽柄与块级弹出菜单交互细节
1. **激活方式**：当鼠标悬停在富文本编辑区的任一文本块/段落上时，在该块的左侧边缘（通常是 margin 区域）动态显示一个轻量化的拖拽柄 `::` 图标。
2. **拖拽操作**：长按该拖拽柄，用户可以上下拖动整块，并在释放时重排内容顺序。
3. **点击操作（块菜单弹出）**：用鼠标单击该拖拽柄，即时弹出一个悬浮的块级控制下拉菜单。
   - **定位机制**：菜单必须使用相对视口的 `fixed` 定位，以防止在 Note Drawer 或其他局部滚动区域中被 `overflow: hidden` / `overflow: auto` 样式所裁剪。位置根据鼠标点击的 Drag Handle 视口坐标动态计算。当菜单底端超出视口下边界时，向上方（Drag Handle 的上方）翻转弹出；若右侧超出视口右边界，则向左平移对齐。
   - **相关功能的底层实现规格**：
     - **块级转换与结构调整**：点击时通过 `setNodeSelection(pos)` 锁定被拖拽块的节点位置，然后再触发 `setParagraph`、`toggleHeading`、列表或引用转换命令。
     - **文字颜色与高亮设置**：通过 `setTextSelection(from, to)` 精准划定当前拖拽块内部文本的起止坐标，再应用 `setColor` 和 `toggleHighlight` Marks。
     - **块级复制/剪切/删除**：通过 `setNodeSelection(pos)` 对块进行整体选中，然后通过调用系统剪贴板指令或 `deleteSelection()` 准确作用于该块。
   - **菜单结构**：
     - **首行（块级标签快捷转换）**：
       - 正文图标：[T]
       - 标题级别：[H1], [H2], [H3], [H4], [H5], [H6]
     - **第二行（内容结构与常用格式）**：
       - 列表转换：[有序列表 1/2], [无序列表], [待办列表]
       - 结构组件：[引用 Quote], [高亮 Highlight], [代码块 CodeBlock]
     - **第三部分（二级子菜单与基本命令）**：
       - **段落颜色**：点击展开常用文本颜色和背景颜色格子（红、黄、蓝、绿等）。
       - **缩进和对齐**：点击展开子菜单选择（左/中/右/两端对齐、增加/减少缩进）。
       - 分割线
       - **复制** / **剪切** / **删除**（以危险红色文字突出显示）。

## 6. 技术规格说明

### 6.1 前端核心组件列表
1. **`DailyReviewPanel.tsx`**：主入口容器，同步数据库，向下分发 `reviews` 数据和当前选中的 `selectedDate`。
2. **`CompoundStats.tsx`**：接收复盘列表，进行打卡天数计算，并渲染 `DayPicker`。
3. **`ReviewEditor.tsx`**：内置 TipTap 实例（`useEditor`），绑定 `onUpdate` 事件，并通过 React Effect 驱动 500ms 防抖更新。
4. **`SimpleEditor.tsx`**：通用 TipTap 编辑器组件，增加 `containerWidth` 监测与 `isCompact` 状态判定，实现自适应工具栏的动态流转和释放。
5. **`BlockDragHandleMenu.tsx`**：挂载 TipTap 官方 `@tiptap/extension-drag-handle` 拖拽柄，并在被点击时渲染包含双行快捷键和列表命令的高级块级控制浮层菜单。

### 6.2 数据模型定义 (参考)
```typescript
export interface DailyReview {
  id: string;      // 格式: review-YYYY-MM-DD
  date: string;    // YYYY-MM-DD
  content: string; // HTML 格式的 TipTap 复盘文本
  rating?: number; // 1-5 的分数值
  createdAt: number;
  updatedAt: number;
}
```

### 6.3 乐观更新与防抖实现说明
- 前端内存乐观更新：当富文本编辑内容或评分改变时，即时更新对应的 UI 状态（如打卡日历点高亮与连续打卡天数）。
- 防抖时间设置：编辑器内容改变时启动 `500ms` 的计时器防抖，避免每次击键都保存；如果在计时器未到时用户直接切换了日期，系统会取消当前防抖计时并**立刻强制执行**当前已修改的数据同步，以防止未保存的内容丢失。
